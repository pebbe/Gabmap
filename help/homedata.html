File format

The data needs to be uploaded in a file with a single table. Each line in the file is a row in the table. Table cells are separated by a tab character. There are no quotes around the contents of a cell, and the contents does not contain escape characters. I.e.: the content of each cell is used at it is.

Lines starting with a hash (#) are considered comments, and ignored.

The first row in the table contains the labels for the items.

The first column in the table contains the names of the locations.

Each column (except column one) has all the variants of a single item for all locations.

Each row (except row one) has the variants of all items for a single locations.

# this is a comment: ignored
item 1item 2item 3
place Avariant A1variant A2variant A3
place Bvariant B1variant B2variant B3
place Cvariant C1variant C2variant C3
place Dvariant D1variant D2variant D3

The first cell in the first row can be empty or missing. When missing, the first line has one cell less than all the other rows.

A cell may be empty for missing data.

When more than one variant exists for one item and one location, put both variants in a single cell, separated by these three characters: space slash space.

variant B2i / variant B2j

Note that all leading and trailing spaces are ignored.

File encoding

The following file encodings are recognised:
  • US-ASCII
  • UTF-8 with/without BOM
  • UTF-16 with BOM
All other encodings are assumed to be ISO-8859-1.
